🔍 Filter Commands

Text filtering, searching, and processing commands — the most essential tools for working with files and data streams. Each example runs on real predefined files in the terminal.

📁 Predefined files available in the terminal:
~/ (home) ├── logs/ │ ├── access.log — web server requests (IP, method, path, status) │ └── error.log — application error messages ├── employees.csv — name,dept,salary,city ├── fruits.txt — list of fruit names (with duplicates) ├── names.txt — sorted list of names ├── names2.txt — another sorted name list (for comm/diff) ├── numbers.txt — one number per line ├── config.txt — key=value config file with comments └── README.md — description of all files

💻 Interactive Terminal

A real simulated Linux terminal. Predefined files are already loaded — try filter commands like grep "error" logs/error.log or cut -d, -f1 employees.csv

student@linuxpath:~
Tips: Use ↑↓ arrows for history · Tab to autocomplete · Try: ls, cat fruits.txt, grep "Alice" employees.csv, cut -d, -f1,2 employees.csv, diff names.txt names2.txt
